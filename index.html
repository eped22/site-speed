Performance Optimization

Statistics

To retain the most visitors, a page should load in 2 seconds or less.

According to a 2006 study by Amazon, each additional 100ms added to load time costs them 1% in sales. For context, a slowdown of 100ms over the course of a day would cost Amazon around $10.4 million.

A Google study found that the probability of page bounce rate increases 32% as mobile load time goes from 1 to 3 seconds.

Site Audit

The first step to improving performance is running an audit. This will give you an initial baseline to refernce going forward. Additionally most audit tools will give you actional tips for improving performance.

Chromium browsers have a built in audit tool called Lighthouse. To use Lighthouse first inspect the page, then click the Audits tab. It may be hidden behind the More Panels More Panels button.

Additional audit tools include PageSpeed Insights and Website Speed Test

Strategies

Improving site performance can feel like an overwheling task. Knock out the quick wins first and then focus on incremental improvement overtime.

  • Reduce # of requests
  • Remove unused code
  • Optimize images
  • Minify CSS and JS
  • Lazy/Defer loading
  • Reduce render blocking
  • Gzip compression
  • Utilize a CDN
  • Leverage caching
  • Reduce fonts & plugins

Tooling

The Coverage Tab

The Coverage tab in Chromium DevTools can help you find unused JavaScript and CSS code.
